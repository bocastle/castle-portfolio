# 포트폴리오 운영 준비 회고

## 세 줄 요약

- 포트폴리오 유입 분석을 위해 GTM/GA4 이벤트 추적을 정리하고, 이력서 링크 유입을 UTM으로 구분할 수 있게 준비했다.
- AdSense는 홈 첫인상에 영향을 주지 않도록 블로그 영역에만 배치했고, 사이트 검토 요청까지 완료했다.
- 코드 검증, Vercel 배포 확인, GA4/AdSense 콘솔 상태 확인을 마쳤으며 남은 항목은 Google 심사 대기다.

## 배포 브랜치와 커밋

- 브랜치: `main`
- 주요 커밋:
  - `142d961 feat: strengthen portfolio proof and analytics`
  - `fade0af Refine blog AdSense placement`
- 공개 URL: `https://bocelog.vercel.app`

## 배포 목적

이 배포의 목적은 포트폴리오를 단순 정적 소개 페이지에서 운영 가능한 개인 서비스에 가깝게 정리하는 것이다. 이력서를 본 사용자가 사이트에 들어왔을 때 어떤 흐름으로 콘텐츠를 탐색하는지 익명 이벤트 단위로 확인하고, 블로그 영역에는 향후 수익화가 가능한 광고 기반을 마련했다.

다만 포트폴리오 홈은 채용 담당자와 면접관의 첫인상이 중요한 화면이므로 광고를 노출하지 않았다. 광고는 블로그 목록, 카테고리, 글 상세처럼 콘텐츠 소비 목적이 분명한 화면에만 배치했다.

## 주요 변경사항

### 분석 이벤트 정리

- GTM을 통해 GA4 기본 태그와 포트폴리오 커스텀 이벤트 태그를 게시했다.
- 페이지 이동, 이력서/연락 CTA, 블로그 글 클릭, 프로젝트 문서/이미지 클릭, 외부 링크 클릭, 스크롤 깊이, 체류 시간성 이벤트를 수집하도록 정리했다.
- GA4에는 주요 이벤트 파라미터를 커스텀 정의로 등록했다.
- 이벤트는 익명 행동 분석 목적이며, 이름/이메일 같은 개인정보 식별 정보는 수집하지 않는다.

### 광고 준비

- AdSense 계정 재활성화 후 `bocelog.vercel.app` 사이트 검토 요청을 완료했다.
- `ads.txt`는 공개 URL에서 정상 응답하도록 확인했다.
- 광고 스크립트와 슬롯은 블로그 레이아웃에서만 로드한다.
- AdSense가 `unfilled` 상태를 반환하거나 처리 완료 후 iframe을 렌더링하지 못하면 빈 광고 공간을 숨기도록 처리했다.
- 광고 계정, 측정 ID, 퍼블리셔 ID 같은 값은 환경변수와 외부 콘솔에 두고 문서에는 기록하지 않는다.

### 포트폴리오 증거 보강

- 프로젝트 카드와 작업 이력에는 운영/성능/분석/광고 준비처럼 실제 서비스 운영에 가까운 경험이 드러나도록 정리했다.
- 모바일 메뉴와 블로그 반응형 문제를 수정했다.
- 블로그 하단 이전/다음 게시글 링크가 작은 화면에서 넘치지 않도록 개선했다.

- 홈 화면은 포트폴리오 소개, 프로젝트, 작업 이력, 블로그 진입 흐름을 유지한다.
- 블로그 화면에서는 광고 준비 코드가 로드되지만 홈에는 로드되지 않는다.

- AdSense 사이트 상세 화면에서 `bocelog.vercel.app`은 검토 요청 후 `준비 중` 상태로 전환됐다.

## 발생한 문제와 대응

AdSense 광고가 보이지 않는 원인을 확인하는 과정에서 코드가 빠진 문제가 아니라 계정 비활성화와 사이트 검토 미완료가 주요 원인임을 확인했다. 계정 재활성화 후 사이트 검토 요청을 완료했고, 코드상 광고 슬롯은 홈이 아니라 블로그에만 남기는 방향으로 정리했다.

AdSense 콘솔의 `Ads.txt 상태`는 한때 `찾을 수 없음`으로 표시됐지만, 실제 공개 URL에서는 `ads.txt`가 정상 응답했다. 콘솔 반영 지연 가능성이 있으므로 심사 상태와 함께 추후 재확인한다.

모바일 글 상세 QA 중 AdSense가 슬롯 처리를 완료했지만 iframe을 렌더링하지 못해 빈 375px 박스가 남을 수 있음을 확인했다. 승인 전 방문자의 읽기 흐름을 해치지 않도록 처리 완료 후 iframe이 없는 슬롯도 빈 광고로 판단해 숨긴다.

## 배운 점

- 분석 도구는 “개인 식별”이 아니라 “익명 행동 흐름”을 보는 도구로 설계해야 한다.
- 이력서 유입은 PDF나 메신저 앱에서 referrer가 사라질 수 있으므로 UTM 링크를 사용하는 편이 안전하다.
- 포트폴리오 홈의 광고 노출은 첫인상을 해칠 수 있어 블로그 영역으로 분리하는 편이 낫다.
- AdSense 문제는 코드, `ads.txt`, 사이트 승인, 계정 상태를 분리해서 확인해야 빠르게 원인을 좁힐 수 있다.

## 다음 개선점

- 이력서 PDF의 포트폴리오 링크를 UTM 링크로 교체한다.
- GA4 보고서에서 `resume` 유입과 포트폴리오 행동 흐름을 확인할 수 있는 탐색 보고서를 저장한다.
- AdSense 심사 결과가 나오면 `준비됨`, `주의 필요`, `검토 실패` 중 어떤 상태인지 확인하고 후속 대응한다.
- 블로그 글 상세에서 광고 위치가 읽기 흐름을 방해하지 않는지 실제 승인 후 다시 확인한다.
