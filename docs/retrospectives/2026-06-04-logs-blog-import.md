# 2026-06-04 logs 블로그 연동 회고

## 배포 내용

`bocastle/logs` 저장소의 대표 글을 포트폴리오 블로그에 공개했다.

이번 배포는 글 원본을 복사하는 방식이 아니라, 포트폴리오에 메타데이터만 두고 상세 본문은 GitHub raw Markdown에서 가져오는 방식으로 구성했다.

## 잘된 점

기존 `api/blog.ts` facade 구조가 있어서 Notion, 로컬 Markdown, logs 글을 같은 UI에 자연스럽게 합칠 수 있었다.

frontmatter가 없는 글도 별도 어댑터를 두면 충분히 공개 가능하다는 점을 확인했다.

카테고리별 썸네일을 먼저 둔 덕분에 모든 글에 개별 이미지를 준비하지 않아도 목록 화면이 빈약해 보이지 않는다.

## 주의할 점

상세 본문은 GitHub raw URL을 fetch하므로, GitHub 접근이 실패하면 해당 logs 글 상세도 실패할 수 있다.

현재는 대표글 21개만 공개한다. 전체 389개를 한 번에 공개하면 중복, 짧은 메모, 알고리즘 풀이가 섞여 포트폴리오 메시지가 흐려질 수 있다.
