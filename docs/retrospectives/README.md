# 배포 회고 문서

## 세 줄 요약

- 배포 이후 회고는 이 폴더에 Markdown으로 남깁니다.
- 문서는 한글로 작성하고, 상단에 세 줄 요약을 둡니다.
- 배포 내용, 검증 결과, 문제 여부, 배운 점, 다음 개선점을 한 문서 안에 정리합니다.

## 목적

이 폴더는 `dev`에서 검증된 기능을 `main`에 병합하고 배포한 뒤, 배포 결과와 회고를 남기는 공간입니다.

회고 문서는 다음 배포자가 단독으로 읽어도 당시 배포 맥락을 이해할 수 있게 작성합니다.

## 파일명 규칙

```text
YYYY-MM-DD-topic.md
```

예시:

```text
2026-06-02-portfolio-projects-release.md
2026-06-05-github-blog-source-release.md
```

## 문서에 포함할 내용

- 세 줄 요약
- 배포 브랜치와 커밋
- 배포 목적
- 주요 변경사항
- QA 및 검증 결과
- 배포 후 확인한 화면이나 기능
- 발생한 문제
- 해결 방법
- 배운 점
- 다음 개선점
- 참고 내용

## 작성 규칙

- 한글로 작성합니다.
- 확장자는 `.md`를 사용합니다.
- 문서 상단에는 `세 줄 요약`을 둡니다.
- 다른 문서를 참조하라는 내용만 남기지 않습니다.
- 필요한 참고 내용은 문서 안에 풀어서 정리합니다.
- 비밀값, 토큰, 인증 정보, 개인 계정 정보는 기록하지 않습니다.
