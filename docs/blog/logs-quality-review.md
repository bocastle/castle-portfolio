# GitHub logs 공개 글 품질 검수

## 검수 기준

`bocastle/logs` 글은 포트폴리오에서 채용 근거로 쓰이므로, 공개 메타데이터는 아래 기준을 통과해야 한다.

- 제목은 깨진 문자가 없어야 한다.
- 설명은 30자 이상 120자 이하로, 글의 실무 의미를 바로 설명해야 한다.
- slug는 `/blog/logs-...` 형식의 영문 소문자 URL이어야 한다.
- 카테고리와 태그는 목록/상세/필터 UI에서 읽기 쉬운 표시명으로 보여야 한다.
- 홈과 블로그 상단 추천 글은 운영/배포, 장애 대응, 성능, 풀스택, 품질 검증, AI QA를 균형 있게 보여줘야 한다.
