# 블로그 상단 글과 공개 글 품질 정리 설계

## 목표

홈에서 정리한 글 6개를 `/blog` 목록 상단에서도 먼저 보여주고, 전체 글 목록은 그 아래에서 이어 보이게 한다. 동시에 카테고리/태그는 포트폴리오에 맞게 한글 표시명을 적용하고, `bocastle/logs` 공개 글 21개의 메타데이터 품질을 검수한다.

## 접근

글 카드 데이터는 `FeaturedWriting` 컴포넌트에서 한 번만 관리한다. 홈과 `/blog`는 같은 카드 목록을 재사용하되, `/blog`에서는 `기술 글` / `먼저 읽어볼 글` 문구로 보여준다.

카테고리와 태그의 원본 `name`은 URL과 필터 키로 계속 사용한다. 화면 표시명은 `src/utils/blog-labels.ts`에서만 변환해 `Backend`는 `백엔드`, `DevOps`는 `운영/배포`, `Performance`는 `성능`처럼 보여준다.

`github-logs.ts`의 21개 공개 글은 slug를 유지하고 제목/설명만 공개용 톤으로 검수한다. 품질 기준과 결과는 `docs/blog/logs-quality-review.md`에 남긴다.

## 검증

- `/blog`는 먼저 읽어볼 글 다음 전체 글 구조를 보여야 한다.
- 글 카드 6개 링크는 홈과 `/blog`에서 같은 URL을 가리켜야 한다.
- 카테고리 링크는 한글로 보이되 `/categories/Backend` 같은 canonical URL을 유지해야 한다.
- 태그 필터는 한글로 보이되 canonical tag 객체를 사용해야 한다.
- logs 21개는 깨진 문자, 중복 pageId, 잘못된 slug, 지나치게 짧거나 긴 설명이 없어야 한다.
