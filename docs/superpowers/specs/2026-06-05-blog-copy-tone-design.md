# 블로그 문구 톤 정리 설계

## 목표

블로그 영역의 문구를 채용 키워드 나열처럼 보이지 않게 정리한다. 기술 글 목록은 개발자 블로그에서 흔히 쓰는 짧고 담백한 표현을 사용하고, 설명 문장은 글의 주제와 기록 맥락만 말한다.

## 적용 방향

- 홈의 글 섹션은 `최근에 정리한 글`로 두고, 설명은 `서비스를 만들고 운영하면서 겪은 문제와 해결 과정을 기록했습니다.`로 정리한다.
- `/blog` 상단은 `기술 글` / `먼저 읽어볼 글`로 보여준다.
- `/blog`의 전체 목록 제목은 `전체 글` / `글 목록`으로 단순화한다.
- 역량 증명 목적을 노골적으로 설명하는 문구는 화면에서 사용하지 않는다.

## 검증

- 홈과 `/blog`에서 새 문구가 렌더링되어야 한다.
- 기존 상단 글 6개 링크는 유지되어야 한다.
- 브라우저 QA는 새 제목인 `먼저 읽어볼 글`, `글 목록`을 기준으로 통과해야 한다.
