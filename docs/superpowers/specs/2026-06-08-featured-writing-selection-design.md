# 상단 글 선별 조정 설계

## 목표

홈과 `/blog` 상단 6개 글이 AI 활용보다 실무형 풀스택 경험을 먼저 보여주도록 조정한다.

## 방향

기존 상단 글의 `Codex로 포트폴리오 QA 자동화하기`는 전체 블로그 목록에 유지하되 상단에서는 내린다. 대신 `React Query 서버 상태 관리`를 올려 프론트엔드 상태 관리 경험을 보강한다.

최종 상단 묶음은 운영/배포, 장애 대응, DB 성능, 프론트 상태 관리, 풀스택 캐시, 테스트 흐름으로 구성한다.

## 검증

- 홈과 `/blog` 상단 글은 6개를 유지한다.
- `React Query 서버 상태 관리` 링크는 `/blog/logs-react-query-server-state`를 가리켜야 한다.
- 상단 카드에 `AI QA` 라벨은 보이지 않아야 한다.
- 기존 AI QA 글은 전체 블로그 글로는 계속 접근 가능하다.
