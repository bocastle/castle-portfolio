## 서비스 흐름을 함께 보는 이유

프론트엔드, 백엔드 API, 데이터, 배포 흐름은 따로 움직이지 않는다.

### 확인하는 기준

- 사용자가 실제로 지나가는 화면과 API 계약을 함께 본다.
- 운영 이슈는 로그, DB, 배포 산출물 기준으로 좁힌다.
- AI 도구는 반복 분석과 검증 속도를 높이는 보조 도구로 둔다.
